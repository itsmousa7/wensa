# Discounts Category — Design Spec

**Date:** 2026-05-29  
**Status:** Approved

## Summary

Add a synthetic "Discounts" chip to the horizontal category bar on the home screen. When selected it shows a paginated feed of all places that currently have any active discount (merchant %, hourly, promo-code-backed, or app-wide auto discount). It behaves identically to existing category chips: single-selection, tap-to-toggle, independent of other categories.

---

## 1. Category Bar (`category_bar.dart`)

- `ListView` grows by 1: `itemCount = cats.length + 1`.
- Index `0` → hardcoded Discounts chip.
  - Icon: `CupertinoIcons.tag_fill` at 42×42 (matches existing icon fallback size).
  - Label: `'Discounts'` (EN) / `'خصومات'` (AR).
  - On tap: `ref.read(selectedCategoryProvider.notifier).select(-1)`.
  - Active check: `selectedIndex == -1`.
- Index `i ≥ 1` → `cats[i-1]`, calls `select(i-1)`, active when `selectedIndex == i-1`.
- No Lottie animation for the Discounts chip (static icon only).

---

## 2. `SelectedCategoryProvider` (no changes)

`select(-1)` already works with the existing toggle logic (`if (state == index) state = null`). The provider stores `int?`; `-1` is a valid Dart `int`.

One guard added in `home_page.dart`: change the `selectedCat` lookup to require `selectedIdx >= 0` before indexing into the categories list.

---

## 3. New `DiscountsFeed` notifier (`category_feed_provider.dart`)

A new `@riverpod` class `DiscountsFeed` alongside the existing `CategoryFeed` and `AllPlacesFeed`.

**`loadMore()` logic:**

1. `await ref.read(merchantDiscountsProvider.future)` and `await ref.read(autoDiscountsProvider.future)`.
2. Build eligible sets:
   - `merchantIds`: from `MerchantDiscount`s where `isCurrentlyActive()` + active targeted `AutoDiscount.targetMerchantIds`.
   - `placeIds`: from `MerchantDiscount`s where `!appliesToAllPlaces` (use `placeIds` list) + active targeted `AutoDiscount.targetPlaceIds`.
3. App-wide check: if any active `AutoDiscount` has `scopeType == 'app'` → query all places (skip OR filter).
4. Empty check: if `merchantIds` and `placeIds` are both empty (and no app-wide discount) → set `hasMore: false, items: []`.
5. Supabase query:
   ```
   content.places_mobile
     .select(id, merchant_id, name_en, name_ar, area, cover_image_url, logo_url, is_verified)
     .eq('place_status', 'approved')
     .or('merchant_id.in.(<merchantIds>),id.in.(<placeIds>)')   // omitted if app-wide
     .order('hotness_score', ascending: false)
     .range(from, to)
   ```
6. Pagination: page size 10, same `CategoryFeedState` model.

## 4. `DiscountsFeedSection` widget (`when_no_data_available.dart`)

New widget alongside `CategoryFeedSection`. Backed by `discountsFeedProvider` (no parameters). Same skeleton → error → empty → infinite-list structure.

Empty state copy:
- EN: `'No discounts available right now'`
- AR: `'لا توجد خصومات متاحة حالياً'`

---

## 5. Home page (`home_page.dart`)

Add constant at top of file:
```dart
const int kDiscountsCategoryIndex = -1;
```

Update `selectedCat` derivation:
```dart
final selectedCat = (selectedIdx != null && selectedIdx >= 0 &&
    categories != null && selectedIdx < categories.length)
    ? categories[selectedIdx]
    : null;
```

Add branch in sliver tree (between `CategoryBar` and the existing `selectedCat == null` default feed):
```
if selectedIdx == kDiscountsCategoryIndex
  → _sectionTitle('Discounts' / 'خصومات')
  → DiscountsFeedSection()
else if selectedCat != null
  → existing CategoryFeedSection(...)
else
  → existing default feed
```

Also invalidate `discountsFeedProvider` in `_onRefresh()`.
